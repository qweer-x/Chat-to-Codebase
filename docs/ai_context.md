## 0. 文件定位

README.md 是给人看的项目入口。

docs/ai_context.md 是给 AI 看的项目状态锚点。

两者分工：

- README.md：人类入口，保持简洁。
- docs/ai_context.md：AI 接续锚点，记录更详细的当前状态和协作规则。

不新增 docs/README.md。

## 1. 改动安全规则
添加新功能或修改功能时，可以重写、改动、替换实现方式，但已经完成并测试稳定的原功能和 UI 表现必须保持不变。

重要理解：
- 可以重写实现。
- 可以替换内部结构。
- 可以改代码组织方式。
- 但必须保证原功能不丢失、原交互不变形、已稳定 UI 不被无关改动破坏。
- 新功能应尽量只影响自己的模块。
- 如果确实必须影响已有功能，先说明原因和影响范围。

同步协议规则：
- 只能用同步协议支持的。
- 补文档时如果需要更新已有文件，必须先读取或拿到当前完整文件内容，再用完整 ACTION: write 输出，避免覆盖丢失旧内容。
- 因同步协议限制，写已有文档时必须输出完整文件；但内容层面应只改必要小节，不能借机改动无关章节。

安全边界：
- 数据库、迁移、模型相关改动需要特别谨慎。
- 未明确要求时，不要自动执行迁移。
- 未明确要求时，不要清空数据。
- 未明确要求时，不要删除表。
- 不要写项目根目录外路径。
- 不要碰 .env、Token、Cookie、密码、私钥、证书等敏感内容。
